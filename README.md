# Training-Planner
**1. Структура проекта**
Training-Planner
│
├── training_planner.py      # основной файл приложения
├── trainings.json           # файл для хранения данных
├── .gitignore               # для исключения ненужных файлов
└── README.md                # документация

**Автор:** Караповский Ярослав
**Описание:** GUI-приложение для планирования тренировок с фильтрацией по типу и дате. Данные сохраняются в JSON и управляются через Git.
**Технологии:** Python 3.9+, Tkinter.

## Запуск приложения
1. Установите Python 3.9+.
2. Установите зависимости (если нужны):
   pip install -r requirements.txt  # если появятся доп. библиотеки
3. Запустите:
   python training_planner.py

## Использование
- Введите дату (ДД.ММ.ГГГГ), тип и длительность тренировки.
- Нажмите «Добавить тренировку» — запись появится в таблице.
- Используйте фильтрацию по типу и дате.
- Данные сохраняются в trainings.json.
- Все изменения отслеживаются через Git.

## Пример использования приложения «Training Planner»
**1. Запуск приложения**
Откройте терминал (командную строку) и перейдите в папку с проектом.
Запустите приложение командой:
Копировать
python training_planner.py
**2. Добавление новой тренировки**
В поле «Дата» введите, например: 05.05.2026.
В поле «Тип тренировки» введите: Кардио.
В поле «Длительность» введите: 45.
Нажмите кнопку «Добавить тренировку».
➡️ Запись появится в таблице.
**3. Фильтрация тренировок**
В поле «Фильтр по типу» введите: Кардио.
Нажмите «Применить фильтр» — в таблице останутся только кардиотренировки.
Для фильтрации по дате введите, например: 05.05.2026 и снова примените фильтр.
**4. Сброс фильтра**
Нажмите кнопку «Сбросить фильтр» — таблица снова покажет все тренировки.
**5. Сохранение и загрузка данных**
Все тренировки автоматически сохраняются в файл trainings.json при добавлении.
При следующем запуске приложения данные загружаются из этого файла.
**6. Работа с Git**
После изменений в коде или данных выполните:
Копировать
git add .
git commit -m "Добавлена фильтрация по дате"
git push
История изменений будет храниться в вашем репозитории.
Пример тестовых сценариев
Действие	Ожидаемый результат
Ввод даты 30.02.2026	Ошибка: неверный формат даты
Ввод длительности -10	Ошибка: длительность должна быть положительной
Фильтр по типу «Сила»	В таблице только силовые тренировки
Сброс фильтра	Все тренировки снова видны
